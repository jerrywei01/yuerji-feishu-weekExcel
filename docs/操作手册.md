# 操作手册

## 1. 项目用途

本项目用于把飞书多维表格中的宝宝带养资料，自动生成每周打卡 Excel，并同步为飞书表格链接。

完整流程：

1. 在 `周表格生成` 新建一条任务
2. 关联到 `宝宝信息表` 中对应宝宝
3. 填写 `方案开始日期` 和 `生成周数`
4. 触发 HTTP 请求
5. 服务端读取数据并生成周打卡表
6. 飞书自动回写生成状态和结果链接

## 2. 飞书表结构

### 宝宝信息表

表 ID：`tblDWFTaPVowtziC`

主要字段：

- `妈妈名称/昵称`
- `宝宝名称`
- `宝宝基本信息`
- `宝宝星级`
- `方案`
- `阶段总结`
- `作息表`
- `辅食安排`
- `瑶浴月历`
- `生成日期`
- `记录 ID`

### 周表格生成

表 ID：`tblFS4XZfacEDDUF`

主要字段：

- `宝宝信息`：关联到宝宝信息表
- `记录 ID`
- `方案开始日期`
- `生成周数`
- `生成状态`
- `结果链接`
- `错误信息`
- `最近生成时间`

## 3. 标题规则

所有标题统一为：

`御儿记VIP宝宝带养 · {宝宝名称}专属作息打卡表{星级}`

例如：

`御儿记VIP宝宝带养 · 小满专属作息打卡表⭐⭐⭐⭐`

应用位置：

- Excel 顶部标题
- 生成的本地文件名
- 飞书表格导入名
- 多维表格结果链接文字

## 7. Nginx 反向代理

如果希望通过 `80` 端口访问：

```nginx
server {
    listen 80;
    server_name 你的域名或公网IP;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

启用：

```bash
ln -s /etc/nginx/sites-available/yuerji /etc/nginx/sites-enabled/
nginx -t
systemctl restart nginx
```

此时访问：

```text
http://你的公网IP/api/feishu/generate-weekly-sheet
```

## 8. 飞书自动化配置

### HTTP 请求

- 方法：`POST`
- URL：`http://你的地址/api/feishu/generate-weekly-sheet`
- Header：

```json
Content-Type: application/json
```

- Body：

```json
{
  "record_id": "{{$json.record_id}}"
}
```

### 建议触发条件

在 `周表格生成` 中：

- 当新增记录时
- 或当某个“生成”按钮/状态字段变化时

## 9. 常见问题

### 9.1 `RecordIdNotFound`

说明传入的 `record_id` 不属于 `周表格生成` 当前那张表。

### 9.2 `缺少记录 ID`

说明周表没有正确关联到宝宝信息表，或关联字段未读取到对应记录。

### 9.3 `上游接口返回的不是合法 JSON`

检查：

- `outputs/last-llm-response.txt`
- `outputs/feishu-last-response.txt`

### 9.4 生成失败但本地服务正常

优先检查：

- 飞书应用权限是否开通
- `FEISHU_TARGET_FOLDER_TOKEN` 是否正确
- DeepSeek Key 是否可用
- 宝宝信息表和周表字段名是否与代码一致
